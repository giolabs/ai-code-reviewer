import { Card, CardGrid } from '@astrojs/starlight/components';

```bash
npx -y ai-code-reviewer@latest review-pr
```

<CardGrid>
  <Card title="Multi-provider" icon="puzzle">
    Supports OpenAI, Anthropic, Gemini, and Ollama. Switch providers with one field in `.ai-review.yml` — no workflow changes needed.
  </Card>
  <Card title="Dependency graph context" icon="bars">
    On JS/TS stacks, analyzes imports and callers of changed files and injects the graph into the LLM prompt to catch bugs outside the diff.
  </Card>
  <Card title="Inline comments + summary" icon="comment">
    Posts findings directly on PR diff lines with color-coded severity. Findings outside the diff go to the PR summary.
  </Card>
  <Card title="Structured output" icon="approve-check">
    Never parses raw LLM text. OpenAI uses `response_format: json_schema`; other providers receive explicit formatting instructions.
  </Card>
  <Card title="Auto tech stack detection" icon="magnifier">
    Detects NestJS, Next.js, React, TypeScript, Node, Flutter, and Laravel from `package.json` and loads built-in rules for each.
  </Card>
  <Card title="Custom rules" icon="pencil">
    Drop a `code-review-rules.md` in your repo and reference it from `.ai-review.yml`. Your rules override the built-in ones.
  </Card>
</CardGrid>
