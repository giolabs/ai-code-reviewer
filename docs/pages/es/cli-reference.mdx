# Referencia de CLI

Todos los comandos están disponibles vía `npx @giolabsuy/ai-code-reviewer` o el binario global `ai-code-reviewer`.

## review-pr

Revisa un pull request de GitHub. Debe ejecutarse dentro de un entorno de GitHub Actions.

```bash
npx @giolabsuy/ai-code-reviewer review-pr [opciones]
```

| Opción | Descripción |
|---|---|
| `--dry-run` | Imprime lo que se publicaría sin escribir en GitHub |
| `--min-severity <nivel>` | Sobreescribe `minSeverity` de la config |
| `--save <ruta>` | Guarda un reporte Markdown en archivo |
| `--config <ruta>` | Ruta al archivo de config (por defecto: `.ai-review.yml`) |

**Variables de entorno requeridas:**

| Variable | Descripción |
|---|---|
| `OPENAI_API_KEY` | (o `ANTHROPIC_API_KEY`, `GEMINI_API_KEY`) |
| `GITHUB_TOKEN` | Provisto automáticamente por GitHub Actions |
| `GITHUB_REPOSITORY` | Provisto automáticamente por GitHub Actions |
| `GITHUB_EVENT_PATH` | Provisto automáticamente por GitHub Actions |

**Códigos de salida:**

| Código | Significado |
|---|---|
| `0` | Revisión completada, sin recomendación `request_changes` |
| `1` | Revisión publicada con `request_changes`, o error |

## review-file

Revisa un archivo individual.

```bash
npx @giolabsuy/ai-code-reviewer review-file <ruta> [opciones]
```

| Opción | Descripción |
|---|---|
| `--min-severity <nivel>` | Filtrar hallazgos por severidad |
| `--save <ruta>` | Guardar reporte Markdown en archivo |
| `--config <ruta>` | Ruta al archivo de config |

**Ejemplo:**

```bash
npx @giolabsuy/ai-code-reviewer review-file src/users/users.service.ts
```

## review-diff

Revisa un diff de git local.

```bash
npx @giolabsuy/ai-code-reviewer review-diff [opciones]
```

Se requiere exactamente uno de `--staged` o `--base`.

| Opción | Descripción |
|---|---|
| `--staged` | Revisar `git diff --cached` (cambios en stage) |
| `--base <rama>` | Revisar `git diff <rama>...HEAD` |
| `--min-severity <nivel>` | Filtrar hallazgos por severidad |
| `--save <ruta>` | Guardar reporte Markdown en archivo |
| `--config <ruta>` | Ruta al archivo de config |

**Ejemplos:**

```bash
# Revisar cambios en stage antes de commitear
npx @giolabsuy/ai-code-reviewer review-diff --staged

# Revisar rama de feature completa contra main
npx @giolabsuy/ai-code-reviewer review-diff --base main

# Revisar cambios en stage y mostrar solo major/critical
npx @giolabsuy/ai-code-reviewer review-diff --staged --min-severity major
```

## handle-feedback

Maneja las respuestas de desarrolladores a comentarios inline del bot (`/explain`, `/dismiss`). Debe ejecutarse dentro de un entorno de GitHub Actions disparado por `pull_request_review_comment`.

```bash
npx @giolabsuy/ai-code-reviewer handle-feedback [opciones]
```

Ver [Feedback inline](./handle-feedback) para el workflow completo.

## init

Crea un archivo de config `.ai-review.yml` con todas las opciones documentadas.

```bash
npx @giolabsuy/ai-code-reviewer init
```

Escribe `.ai-review.yml` en el directorio actual. Es seguro ejecutarlo en cualquier repo — no sobreescribe un archivo existente.

## Niveles de severidad

De mayor a menor:

| Nivel | Cuándo se usa |
|---|---|
| `critical` | Vulnerabilidades de seguridad, riesgo de pérdida de datos |
| `major` | Bugs, lógica rota, problemas graves de performance |
| `minor` | Calidad de código, mejoras potenciales |
| `info` | Informativo, buenas prácticas |
| `nitpick` | Preferencias de estilo, cosmético |

El flag `--min-severity` filtra todo lo que está por debajo del nivel especificado. Por ejemplo, `--min-severity major` muestra solo hallazgos `major` y `critical`.
