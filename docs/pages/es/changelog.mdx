# Changelog

## 0.1.0-beta.4

- **Auto-aprobar**: nuevo bloque de config `autoApprove`. Cuando el modelo recomienda `approve`, no hay hallazgos `critical`/`major` y el `overallScore` cumple el `minScore`, el bot publica un evento GitHub `APPROVE` real. Opt-in — deshabilitado por defecto. Ver [Auto-aprobar](./auto-approve).
- **CI: auto-update del dist-tag `latest`** después de cada publicación beta en `publish.yml`.
- **Fix**: el workflow `handle-feedback` ahora usa el dist-tag `@beta` para evitar traer una versión desactualizada durante el ciclo beta.

## 0.1.0-beta.3

- **Feedback inline (`handle-feedback`)**: los desarrolladores pueden responder `/explain` para obtener una explicación detallada de un hallazgo, o `/dismiss` para marcarlo como resuelto. Requiere el job del workflow `handle-feedback` y `feedback.enabled: true` en la config. Ver [Feedback inline](./handle-feedback).
- **Auto-resolución en push**: cuando se suben nuevos commits que abordan un hallazgo, el bot resuelve el comentario automáticamente.

## 0.1.0-beta.2

- **Multi-proveedor**: soporte agregado para Anthropic (modelos `claude-*`) y Gemini (modelos `gemini-*`). Configurar con `provider: anthropic` o `provider: gemini` en `.ai-review.yml`.
- **Ollama**: soporte agregado para proveedor auto-hospedado. Configurar con `provider: ollama` y `model: codellama` (o cualquier modelo descargado). `ollamaUrl` por defecto es `http://localhost:11434`.
- **Grafo de dependencias** (solo JS/TS): el revisor ahora importa dependencias de 1 nivel de los archivos modificados para detectar riesgos de regresión.
- **Comando `init`**: `npx @giolabsuy/ai-code-reviewer init` genera un `.ai-review.yml` pre-completado.

## 0.1.0-beta.1

Release beta inicial.

- Comando `review-pr` — integración con GitHub Actions
- Comando `review-file` — revisión local de archivo individual
- Comando `review-diff` — revisión local de diffs (`--staged` o `--base`)
- Proveedor OpenAI con output estructurado (`json_schema`)
- Auto-detección de stack: NestJS, Next.js, React, TypeScript, Node.js, Flutter, Laravel, genérico
- Templates de reglas built-in por stack
- Reglas personalizadas vía `rules:` o `customInstructions:` en la config
- Comentarios inline en el diff del PR + comentario resumen
- Filtrado por severidad (`minSeverity`)
- Código de salida `1` en `request_changes`
- Exportación de reporte Markdown con `--save <ruta>`
- Output bilingüe: `language: es | en`
